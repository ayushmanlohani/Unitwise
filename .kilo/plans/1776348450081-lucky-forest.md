## Option 2: Context Validation (To Implement)

### Approach
Check if retrieved context is actually relevant to the query before answering. If context is too short or doesn't match query keywords, reject.

### Implementation Steps

**Step 1: Modify `searcher.py`**
- Lower threshold from 1.3 to **0.8** for stricter matching
- Add check: if multiple top results have high scores, reject

**Step 2: Modify `answerer.py`**
- Add context length check - reject if combined context is too short (e.g., < 100 chars)
- Add keyword overlap check - if query keywords don't match any context, reject
- Make rejection message consistent
